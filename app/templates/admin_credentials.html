Linux root password
—
Console login only — sshd is PermitRootLogin prohibit-password.
PostgreSQL meridian role password
—
App connects as this role. Rotation updates /etc/meridian/meridian.conf automatically; the new DSN takes effect at next meridian-app restart.
After rotation you'll see a "restart required" note. Restart meridian-app at a convenient time — the live session keeps working on the old connection until then.
Row-HMAC key (tamper-evident chain)
—
File: /etc/meridian/secrets/row_hmac.key
DESTRUCTIVE. Every existing row_hash in audit_events, cert_events, approvals, and the other tamper-evident tables becomes invalid. You must run Integrity → Rebaseline chain immediately after this rotation or the portal will alarm on every subsequent scan.
Why rotate: golden-image cloning, insider-risk response, or compliance-driven key rollover.
Master key (secrets vault)
—
File: /etc/meridian/secrets/master.key
DESTRUCTIVE. Every field-encrypted secret (SMTP credentials, LDAP bind password, SNMPv3 users, webhook HMAC secrets, cloud-API keys) is encrypted under the OLD key and will become unreadable. You must re-enter each one via Admin → Integrations after this rotation.
SSH host keys
—
Files: /etc/ssh/ssh_host_*. Old keys archived under /etc/ssh/archived_hostkeys/.
After rotation, every existing SSH client will show WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED. Clear the old fingerprint with ssh-keygen -R <host> before reconnecting.
