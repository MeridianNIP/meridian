Scope is a target allow-list, not a role system. This page controls which networks sandboxed tools (ping, traceroute, dig, port scan, pcap, HTTP test) are allowed to query — it's a firewall in front of the toolbox. It does not govern who can log in or what each user can do. For roles, permissions, and account management see Admin → Users.

How classification works
  1. Deny rules run first — a host inside any deny CIDR is rejected regardless of scope or role.
  2. internal_extra / external_extra overlays override the RFC1918 default. Use these for CGNAT (100.64/10), partner ranges, or public IPs that are actually "yours".
  3. Otherwise, RFC1918 / loopback / link-local → internal, everything else → external.
  4. Hostnames that don't resolve to a literal IP are classified as unknown — scope_of_use does not block them, but deny rules still run once the probe resolves.
