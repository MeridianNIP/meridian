What outbound webhooks are for
Push Meridian events into external systems (PagerDuty, Slack incoming webhooks, ServiceNow, Splunk HEC, a SIEM). Every POST is JSON and HMAC-signed; the receiver should verify the X-Meridian-Signature header before acting. Common use cases: opening a page when a monitor goes down, appending an audit row to a SIEM, kicking a CI pipeline when a runbook finishes.
Sample payload + signature verification 
POST https://events.pagerduty.com/v2/enqueue
Content-Type: application/json
X-Meridian-Event: monitor.incident.opened
X-Meridian-Delivery: 8c4d9b2e-...
X-Meridian-Signature: sha256=a73c9f... (HMAC-SHA256 over the raw body)

{
  "event": "monitor.incident.opened",
  "ts": "2026-04-19T05:00:00Z",
  "monitor": { "id": "...", "target": "api.acme.com", "kind": "https" },
  "severity": "down",
  "detail": { "http_status": 503, "rtt_ms": 8210 }
}

# Python receiver — verify before trusting:
import hmac, hashlib
mac = hmac.new(signing_secret, raw_body, hashlib.sha256).hexdigest()
if not hmac.compare_digest(f"sha256={mac}",
        request.headers["X-Meridian-Signature"]):
    return 401

What inbound receivers are for
Accept signed POSTs from external systems and act on them: auto-create a monitor when a new service is registered in your CMDB, open an approval when GitHub posts a push event to a protected repo, create a ticket when an IDS tool raises an alert. Each receiver gets its own URL and HMAC signing secret; callers must send the signature header or the request is rejected.
Sample inbound POST + curl example 
BODY=$(cat <<'JSON'
{"event":"service.added","service":"payments-api","tier":"critical"}
JSON
)
SIG=$(printf '%s' "$BODY" | openssl dgst -sha256 -hmac "$SIGNING_SECRET" -r | awk '{print $1}')

curl -X POST https://meridian.example.com/api/v1/webhooks/inbound/cmdb-sync \
  -H "Content-Type: application/json" \
  -H "X-Meridian-Signature: sha256=$SIG" \
  --data "$BODY"
New inbound receiver
After creation you'll receive a unique URL and a signing secret. The sender must set X-Meridian-Signature: sha256=<hex> (HMAC-SHA256 of the raw body, computed with the secret). Unsigned or mismatched requests are rejected.
